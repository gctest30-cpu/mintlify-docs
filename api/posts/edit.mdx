Use this endpoint to update an existing post. You can modify any combination of the post's title, content, and visibility. Only the fields you include in the request body will be changed — omitted fields are left as they are. You must be the owner of the post to edit it; the `{user_id}` in the path must match your authenticated identity.

## Endpoint

```
PUT /{user_id}/{post_id}/edit-post
```

## Authentication

<Warning>
  All requests to this endpoint must include a valid `Authorization: Bearer <token>` header. Requests without a valid token will receive a `401 Unauthorized` response. You may only edit posts that belong to your own account.
</Warning>

## Path Parameters

<ParamField path="user_id" type="string" required>
  The UUID of the authenticated user who owns the post. Must match the identity in your Bearer token.

  **Example:** `e3b7c2a1-4f9d-4b8e-9c3a-1d2e5f6a7b8c`
</ParamField>

<ParamField path="post_id" type="string" required>
  The UUID of the post you want to edit.

  **Example:** `a1b2c3d4-e5f6-7890-abcd-ef1234567890`
</ParamField>

## Request Body

All body fields are optional. Include only the fields you wish to update.

<ParamField body="title" type="string">
  The updated title for the post. Maximum length is **128 characters**. Pass an empty string or omit entirely to leave the current title unchanged.
</ParamField>

<ParamField body="content" type="string">
  The updated body text for the post. Maximum length is **1024 characters**. Omit to leave the existing content unchanged.
</ParamField>

<ParamField body="visibility" type="string">
  The updated visibility setting for the post. Controls who can see the post going forward.

  **Example values:** `"public"`, `"friends"`, `"private"`
</ParamField>

## Request Example

<CodeGroup>

```bash cURL
curl --request PUT \
  --url "https://api.nsn.com/e3b7c2a1-4f9d-4b8e-9c3a-1d2e5f6a7b8c/a1b2c3d4-e5f6-7890-abcd-ef1234567890/edit-post" \
  --header "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  --header "Content-Type: application/json" \
  --data '{
    "title": "My updated post title",
    "content": "I have updated this post with some new and improved content.",
    "visibility": "friends"
  }'
```

```python Python
import httpx

USER_ID = "e3b7c2a1-4f9d-4b8e-9c3a-1d2e5f6a7b8c"
POST_ID = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"

response = httpx.put(
    f"https://api.nsn.com/{USER_ID}/{POST_ID}/edit-post",
    headers={"Authorization": "Bearer YOUR_ACCESS_TOKEN"},
    json={
        "title": "My updated post title",
        "content": "I have updated this post with some new and improved content.",
        "visibility": "friends",
    },
)
print(response.json())
```

</CodeGroup>

## Response

### 200 OK

Returned when the post is successfully updated.

```json
{
  "message": "Post edited successfully."
}
```

<Note>
  After a successful edit, the post's `updated_at` timestamp is refreshed to reflect the time of the most recent change. You can verify this by calling `GET /{user_id}/find-post`.
</Note>

## Error Responses

| Status | Meaning | When it occurs |
|--------|---------|----------------|
| `401 Unauthorized` | Authentication failed | The `Authorization` header is missing, the token is invalid/expired, or you do not own the post |
| `500 Internal Server Error` | Unexpected server error | An unhandled exception occurred while processing the request |
