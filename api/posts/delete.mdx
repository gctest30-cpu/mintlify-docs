Use this endpoint to remove a post from your profile and all feeds. Calling this endpoint immediately hides the post from view across NSN. You must be the owner of the post to delete it; the `{user_logged}` in the path must match your authenticated identity.

## Endpoint

```
DELETE /{user_logged}/{post}/delete-post
```

## Authentication

<Warning>
  All requests to this endpoint must include a valid `Authorization: Bearer <token>` header. Requests without a valid token will receive a `401 Unauthorized` response. You may only delete posts that belong to your own account.
</Warning>

## Path Parameters

<ParamField path="user_logged" type="string" required>
  The UUID of the authenticated user who owns the post. Must match the identity in your Bearer token.

  **Example:** `e3b7c2a1-4f9d-4b8e-9c3a-1d2e5f6a7b8c`
</ParamField>

<ParamField path="post" type="string" required>
  The UUID of the post you want to delete.

  **Example:** `a1b2c3d4-e5f6-7890-abcd-ef1234567890`
</ParamField>

## Request Example

<CodeGroup>

```bash cURL
curl --request DELETE \
  --url "https://api.nsn.com/e3b7c2a1-4f9d-4b8e-9c3a-1d2e5f6a7b8c/a1b2c3d4-e5f6-7890-abcd-ef1234567890/delete-post" \
  --header "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```python Python
import httpx

USER_LOGGED = "e3b7c2a1-4f9d-4b8e-9c3a-1d2e5f6a7b8c"
POST = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"

response = httpx.delete(
    f"https://api.nsn.com/{USER_LOGGED}/{POST}/delete-post",
    headers={"Authorization": "Bearer YOUR_ACCESS_TOKEN"},
)
print(response.json())
```

</CodeGroup>

## Response

### 201 Created

Returned when the post is successfully deleted.

```json
{
  "message": "Post deleted successfully."
}
```

<Note>
  Once deleted, the post is immediately removed from your profile and all feeds across NSN. If you require permanent deletion of your data, please refer to the account data removal process in the [Privacy & Data](#) section.
</Note>

## Error Responses

| Status | Meaning | When it occurs |
|--------|---------|----------------|
| `401 Unauthorized` | Authentication failed | The `Authorization` header is missing, the token is invalid/expired, or you do not own the post |
| `500 Internal Server Error` | Unexpected server error | An unhandled exception occurred while processing the request |
